# Actividad 1: Retrospectiva del Equipo

Se realizó una retrospectiva utilizando la dinámica **Start – Stop – Continue**.

## Resultados

### Start (Comenzar)

* Implementar revisiones de código más frecuentes
* Documentar mejor los endpoints de la API
* Definir tareas más pequeñas y claras

### Stop (Dejar de hacer)

* Subir cambios sin pruebas previas
* Trabajar directamente sobre ramas principales
* Falta de comunicación en el progreso de tareas

### Continue (Continuar haciendo)

* Uso de commits con convención
* Trabajo colaborativo en GitHub
* Uso de Pull Requests para integración

---

## Action Items priorizados

Mediante votación, se definieron los siguientes 3 más importantes:

1. Mejorar la documentación del proyecto (README y API)
2. Aplicar revisiones de código obligatorias
3. Dividir tareas grandes en issues más pequeños
